## META-FINDING (2026-06-25 — THE LIMIT MOVED FROM CAPACITY TO TRAINABILITY)
**The biggest shift in the investigation. After the in-channel scalar holographic limit was confirmed (8.89%, capacity-bound), the bounded MATRIX memory (DeltaNet) line changed the character of the wall entirely.**

**The bounded DeltaNet fast-weight matrix CAN beat the holographic ceiling — when it trains.** `M_t = M_{t-1}(I−βkkᵀ)+βvkᵀ`, a real D×D matrix per head (verified bounded: n_heads·d_k·d_v fixed, T- and vocab-independent, NOT the alphabet-indexed cheat). When it ignites: **12.21% / 9.42%** (seeds 1, 123) vs holo 7.28% same-run. The (I−βkkᵀ) projector erases the new key's subspace before writing, so distinct keys don't superpose → no 1/√N law in principle.

**BUT the limit is no longer representation OR capacity — it is OPTIMIZATION ONSET (trainability).** On ~⅔ of seeds the model NEVER finds the erase-then-write mechanism: loss FREEZES at exactly **ln(64)=4.159** (the uniform-over-64-values fixpoint — the model outputs the marginal and learns nothing). Ignition rate measured at **2/6 (33%) for d_k=32, 0/3 for d_k=64**. This is NOT undertraining — a learning model's loss falls; these are flat from step 500 through 4500. The mechanism is a sharp, hard-to-find minimum that some inits fall into and most miss.

This **sharpens, one level deeper, the "optimization not representation" framing**: the question was thought to be "how do you WRITE separably" (now answered — the matrix erase does it); the real question is **"does the optimizer FIND the write."** Trainability, not capacity, not representation.

**What's dead as an onset lever:** γ_t matrix decay (entry 16, erases everything), bigger d_k (entry 17, harder landscape), unscaled rectified-power readout (entry 18, crushes channels). **What stays open:** how to make the bounded-matrix mechanism ignite RELIABLY — init schemes (β-bias high, spectral W_k init, QK-norm), curriculum (warm up at n_pairs=2), escape from the ln(vocab) plateau, chunk-parallel DeltaNet for conditioning, or a small attention warm-up head that bootstraps the onset then detaches. A learned-decorrelating-read (mmse) matrix is the clean test of whether a learned read matrix (vs the broken unscaled poly) lifts the holographic baseline.

## Process breadcrumbs (so we don't repeat)
- ALWAYS pin readout = tanh_m for ≤1500-step cheap sweeps (the rms confound bit us once).
- ALWAYS verify the K=1 / baseline arm reproduces ~9% before trusting the swept arms — a dead baseline = a confounded sweep.
- Multi-seed mandatory; single-seed positives on a chance-flat loss are noise (the seed-42 phase-GSSM false positive).
- Self-test reduction (use_phase=False == Selective, K=1 == baseline) catches mechanism bugs; a passing reduction + chance recall = a RUNNER/config bug, not a mechanism bug.
- **3-seed positives DIE at 5 seeds (the K=2 lesson, 2026-06-25).** A "small real gain" with a std-shrink on n=3 is the classic false positive on a chance-flat loss. NEVER bank a lever before a 5-seed SAME-RUN within-seed-paired re-run. K=2 held credence across two 3-seed runs before the paired 5-seed run killed it.
- **Trust the ⟨s²⟩ instrument before the recall verdict, and check the instrument itself.** The stack harness reported ⟨s²⟩≈1.0019 from a fixed dummy input (BUG); the sweep harness reported 1.6–1.7 from real phi (correct). Read ⟨s²⟩ on actual sequence embeddings, and when an auto-verdict says "need higher λ," check whether higher λ even moves the diagnostic the right way (for β=3 repulsion it does NOT — it overshoots).
